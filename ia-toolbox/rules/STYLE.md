# Rule - Style et conventions

## Nommage

- Classes en PascalCase, methodes/variables en camelCase (Java/TS).
- Entites au singulier (Demande), routes REST au pluriel (/demandes).
- Booleens prefixes est/has (estResolue, hasAssignee).
- Utiliser des noms en francais.

## Format

- Une responsabilite par classe/fonction.
- Pas de nombre magique : constante nommee.
- Commentaires seulement pour le "pourquoi", jamais pour paraphraser le code.

## A ne pas faire

- Pas d'abreviation obscure (dmd, stat).
- Pas de code mort ni de TODO sans ticket associe.
- pas de variable à un caractère.
